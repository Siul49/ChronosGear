# 시계공 히든 설정 (개발자 전용)

> ⚠️ **주의**: 이 문서는 개발팀 전용입니다. 외부 공개 금지.

---

## 1. 시계공의 정체

### 진실
```
시계공 = 무수한 타임라인을 반복한 "다른 크론"

- 수백, 수천 번의 루프를 경험
- 어느 순간 "루프 밖"으로 나가는 방법 발견
- 다락방을 관리하며 "다음 후계자"를 기다림
- 특정 타임라인에 속하지 않음 (초월적 존재)
```

### 목적
- 시간의 흐름을 유지하되, 완전히 붕괴하지 않도록 조율
- "자유의지로 흐름을 선택하는 자"를 후계자로 선택
- 크론이 그 자격을 증명하면 선택권 부여

## 2. 히든 조건 (비공개)

### 필수 조건 4가지

| 조건 | 설명 | 체크 방법 |
|------|------|-----------|
| **기억의 조각 100%** | 모든 30개 수집 | 인벤토리 체크 |
| **개입 최소화** | 레이/벨로/모라 과거에 모두 "내버려둠" | 분기 플래그 |
| **노화 게이지 80%+** | 충분히 늙어야 함 | 노화 수치 체크 |
| **모라 전투 특수 행동** | 최종 보스전에서 특정 순간 공격 안함 | 타이머 체크 |

### 조건 설계 의도
```
"히든 엔딩을 노린다"는 마음으로는 달성 어렵게 설계

1. 개입 최소화 = 친구들을 도와주고 싶은 마음을 참아야 함
   → 대부분의 플레이어는 본능적으로 도와줌

2. 노화 80% = 스킬을 적극적으로 사용해야 함
   → 아끼는 플레이 스타일과 충돌

3. 100% 수집 = 꼼꼼한 탐험 필요
   → 시간 투자

4. 모라 전투 특수 행동 = 공격 본능 억제
   → 모라가 특정 대사를 할 때 3초간 공격 안하면 트리거
```

## 3. 복선 레이어 시스템

### 레이어 0: 조건 0% (기본)
- 시계공에 대한 언급 **완전 없음**
- 플레이어는 트루/배드 엔딩만 인지
- 히든 엔딩 존재 자체 모름

### 레이어 1: 조건 50% 달성
| 장소 | 연출 | 플레이어 인식 |
|------|------|---------------|
| 다락방 | 알 수 없는 그림자 잠깐 스침 | "뭐지?" 정도 |
| 던전 | 이상한 발자국 (조사 불가) | 호기심 유발 |
| 벨로 대화 | "누군가 지켜보는 것 같아" | 복선으로 인식 안함 |

### 레이어 2: 조건 80% 달성
| 장소 | 연출 | 플레이어 인식 |
|------|------|---------------|
| 다락방 구석 | 낡은 로브 조각 발견 | "누구 거지?" |
| 거울 | 잠깐 다른 모습 비침 | "?!" (설명 없음) |
| 모라 대화 | "네가 처음이 아니야" | "무슨 뜻이지?" |

### 레이어 3: 조건 100% 달성
- 4장 기어 파괴 직전 시간 멈춤
- 시계공 등장
- 히든 엔딩 분기 진입

## 4. 시계공 등장 시나리오

### 등장 조건
```
if (기억의조각 == 100 && 
    개입_레이 == false && 
    개입_벨로 == false && 
    개입_모라 == false &&
    노화게이지 >= 80 &&
    모라전투_특수행동 == true) {
    트리거_시계공_등장();
}
```

### 등장 연출
```
[4장: 기어 파괴 직전]

(크론이 기어를 부수려는 순간)
(화면 정지 - 프레임 완전 멈춤)
(색채 반전)

(다락방 구석에서 로브를 입은 노인 등장)

시계공: "잠깐."
시계공: "네가 자격을 증명했구나."

크론: "...누구야?"

시계공: "너. 그리고 나."
시계공: "수천 번의 루프 끝에 여기까지 온 자."
```

### 선택지
```
시계공: "선택해라."
시계공: "평범한 삶으로 돌아갈 것인가,"
시계공: "아니면 새로운 시계공이 될 것인가."

[1] 돌아간다 → 트루 엔딩과 동일
[2] 시계공이 된다 → 히든 엔딩 전용
```
