# 팀 협업을 위한 유니티 개발 환경 구축 가이드

> **Chronos Gear** 프로젝트 팀원을 위한 초기 세팅 매뉴얼입니다.
> 모든 팀원은 이 문서를 따라 개발 환경을 구축해주세요.

---

## 1. 필수 프로그램 설치

### Unity Hub & Editor
1. **[Unity Store](https://store.unity.com/)**에서 Unity Hub 다운로드 및 설치
2. Unity Hub 실행 > `Installs` 탭 > `Install Editor` 클릭
3. **`2022.3.x LTS`** 버전 선택 (팀원 간 마이너 버전 일치 권장, 예: 2022.3.20f1)
4. **Modules** 선택:
   - Microsoft Visual Studio (또는 VS Code 사용 시 생략)
   - Windows Build Support (IL2CPP 포함)
   - Documentation

### Git (버전 관리)
1. **[Git for Windows](https://git-scm.com/download/win)** 설치
   - 설치 시 **Use Git from the Windows Command Prompt** 선택
   - **Checkout as-is, commit Unix-style line endings** 선택 (줄바꿈 충돌 방지)
2. **Git 클라이언트 설치 (선택)**
   - **[GitHub Desktop](https://desktop.github.com/)** (초보자 강추, 무료 & 직관적)
   - **[SourceTree](https://www.sourcetreeapp.com/)** (기능 강력함)
   - **[Fork](https://git-fork.com/)** (빠르고 가벼움)
   - *익숙한 도구가 있다면 그것을 사용해도 됩니다.*

## 2. 프로젝트 생성 (팀장/초기 세팅 담당자)

> 이미 프로젝트가 생성된 경우 **3번(프로젝트 가져오기)**으로 건너뛰세요.

### 2-1. Unity 프로젝트 생성
1. Unity Hub > `Projects` > `New project`
2. **Editor Version**: `2022.3.x LTS` 선택
3. **Template**: **`Universal 2D`** 선택
   - *버전에 따라 `2D (URP)`라고 나올 수도 있습니다. 둘 다 같은 것입니다.*
4. **Project Name**: `ChronosGear` (공백 없이 입력)
5. `Create project` 클릭

### 2-2. 초기 설정 (Project Settings)
1. `Edit` > `Project Settings`
2. **Player**:
   - Company Name: `[YourTeamName]`
   - Product Name: `Chronos Gear`
   - Version: `0.1.0`
   - Color Space: `Linear` (URP 권장)
3. **Editor** > **Asset Serialization**:
   - Mode: `Force Text` (Git 충돌 방지를 위해 필수)

### 2-3. 필수 패키지 설치
1. `Window` > `Package Manager`
2. 좌측 상단 `Packages: Unity Registry` 선택
3. 다음 패키지 검색 및 설치:
   - **Cinemachine** (카메라 시스템)
   - **Input System** (새로운 입력 시스템)
   - **Timeline** (컷신 연출)
   - **2D Sprite** / **2D Tilemap Editor**

## 3. GitHub 프로젝트 세팅 (가장 중요!)

> **Q. 유니티 자체 버전 컨트롤(Plastic SCM)은 안 쓰나요?**
> A. 유니티에 내장된 기능(DevOps)이 있지만, 전 세계 개발 표준인 **Git + GitHub**를 쓰는 것이 훨씬 유리합니다. (무료, 문제 해결 정보 많음, 취업 시 필수 스킬)
> 
> **핵심 흐름**: `내 컴퓨터(로컬)` ↔ `Git 프로그램(중계)` ↔ `GitHub 웹사이트(원격 저장소)`
> *웹사이트에 직접 파일을 올리는 게 아니라, Git 프로그램으로 동기화(Push/Pull) 하는 방식입니다.*

### 3-1. 원격 저장소 만들기 (팀장만 수행)
1. **[GitHub.com](https://github.com/)** 로그인 > `New Repository` 클릭
2. **Repository name**: `ChronosGear`
3. **Add .gitignore**: 반드시 **`Unity`** 검색해서 선택 (★매우 중요★)
   - *이걸 안 하면 잡동사니 파일 수만 개가 올라가서 프로젝트가 터집니다.*
4. `Create repository` 클릭
5. 만들어진 주소(`https://github.com/.../ChronosGear.git`) 복사

### 3-2. 내 컴퓨터로 가져오기 (팀원 공통)
1. Git 클라이언트(GitHub Desktop 등) 실행
2. `Clone Repository` (복제하기) 클릭
3. **URL** 탭에 복사한 주소 붙여넣기
4. **Local Path**: `C:\Projects\ChronosGear` 처럼 찾기 쉬운 경로 선택
5. `Clone` 버튼 클릭 -> 내 컴퓨터에 빈 폴더가 생깁니다.

### 3-3. 유니티 프로젝트 넣기 (초기 세팅)
1. **팀장**이 아까 만든 유니티 프로젝트 폴더(`Assets` 폴더가 들어있는)를 통째로 복사
2. 방금 Clone한 폴더 안에 붙여넣기
3. Git 클라이언트에서 파일들이 감지되면 `Summary` 쓰고 `Commit` -> `Push`
4. **팀원**들은 `Fetch` -> `Pull` 누르면 프로젝트가 짠 하고 받아집니다.

## 6. Mac / Windows 크로스 플랫폼 협업 주의사항

팀 내에 Windows와 Mac 사용자가 섞여 있다면 **반드시** 아래 내용을 확인해야 합니다.

### 6-1. 줄바꿈 문자 (Line Endings) 문제 해결
윈도우는 줄바꿈을 `CRLF`(\r\n)로, 맥은 `LF`(\n)로 처리합니다. 이걸 통일하지 않으면 코드 한 줄만 고쳐도 파일 전체가 바뀐 것으로 인식되어 **무한 충돌**이 발생합니다.

> **해결책**: 프로젝트 최상단에 제가 만들어드린 `.gitattributes` 파일이 이 문제를 자동으로 해결해줍니다.
> (텍스트 파일은 자동으로 통일하고, 이미지/효과음 파일은 건드리지 않도록 설정됨)

### 6-2. 파일 이름 대소문자 (Case Sensitivity)
- **Windows**: `texture.png`와 `Texture.png`를 **같은 파일**로 인식합니다.
- **Mac/Linux**: 둘을 **다른 파일**로 인식합니다.

> **주의**: 코드에서 `Resources.Load("Player")`라고 썼는데 실제 파일이 `player.png`라면, 윈도우에서는 되지만 맥에서는 **빌드 에러**가 납니다.
> **규칙**: 파일명은 항상 **코드와 정확히 대소문자를 일치**시켜 주세요. (가급적 PascalCase 권장)

### 6-3. 한글 파일명 (자소 분리 현상)
맥에서 만든 한글 파일(`안녕.png`)이 윈도우에서 보면 `ㅇㅏㄴㄴㅕㅇ.png` 처럼 자음/모음이 분리되어 보일 수 있습니다. (NFD/NFC 정규화 차이)
> **규칙**: **에셋(이미지, 사운드 등) 파일명은 되도록 '영어'를 사용**하는 것이 정신 건강에 좋습니다. (문서는 한글이어도 괜찮습니다)

### 4-1. 씬(Scene) 작업 규칙
- **메인 씬 동시 수정 금지**: `Level_01` 같은 씬 파일은 충돌 해결이 매우 어렵습니다.
- **프리팹(Prefab) 활용**: 캐릭터, UI, 기믹 등은 반드시 프리팹으로 만들어 개별 수정하세요.
- **전용 테스트 씬**: `Test_KKS`(본인 이니셜) 씬을 만들어 개인 테스트 진행

### 4-2. 메타 파일 (.meta)
- **절대 삭제 금지**: Unity는 파일마다 고유 ID(GUID)를 `.meta` 파일에 저장합니다.
- 파일 이동/이름 변경은 **반드시 Unity 에디터 내부**에서 수행해야 메타 파일이 정상적으로 따라갑니다. (탐색기에서 변경 금지)

### 4-3. 커밋 메시지
- `feat: 플레이어 점프 구현`
- `fix: 벽 통과 버그 수정`
- `art: 주인공 스프라이트 추가`
- `docs: 기획 문서 업데이트`

## 5. 트러블슈팅

**Q. 스크립트를 열었는데 자동 완성이 안 돼요.**
A. `Edit` > `Preferences` > `External Tools` > `External Script Editor`가 Visual Studio(또는 VS Code)로 설정되어 있는지 확인하고 `Regenerate project files` 클릭.

**Q. Git 충돌(Conflict)이 났어요.**
A. 씬 파일이나 `.Yaml` 파일 충돌이라면, 충돌이 난 파일의 `Smart Merge` 툴을 사용하거나, 담당자끼리 소통하여 한쪽의 변경사항을 덮어쓰기(Ours/Theirs) 해야 합니다. (가급적 같은 파일을 동시에 건드리지 마세요!)
